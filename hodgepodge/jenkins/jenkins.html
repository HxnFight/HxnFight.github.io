build category

freestyle

  • 此类别分一下几个步骤,在linux上话,配合shell脚本,就可以轻松的完成项目的部署和发布 所有的逻辑都写在脚本里控制,在general中定义好变量,在build里直接调用变量和预先写好的脚本, 对目标机的管理,可以考虑使用ansible.
  • 缺点就是每次新加项目,需要额外的修改脚本,再个就是比较依赖shell.(windows 用户可以考虑bat,或着poweshell)

         1: echo $deploy_env
 2: case $deploy_env in
 3:     deploy)
 4:         echo "deploy: $deploy_env"
 5:         /data/project/test/script/sw_deploy.sh --local-repository=$opt_local_repo --repository-url=$opt_git_url --backup-dir=$opt_bak_dir --webdir=$opt_web_dir --infodir=$opt_info_dir --version=$opt_version --package=$opt_war_dir
 6:     if [ $? == 0 ];then
 7:             #创建目标本地仓库目录
 8:           ansible test -m file -a "path=$opt_local_repo/target state=directory"
 9:             #创建目标本地信息存放目录
10:       ansible test -m file -a "path=$opt_info_dir state=directory"
11:             #创建目标线上业务目录
12:           ansible test -m file -a "path=$opt_web_dir state=directory"
13:             #创建目标备份目录
14:             ansible test -m file -a "path=$opt_bak_dir state=directory"
15:             #创建目标存放目录,将打包代码发放至目标目录
16:             dir_path=`dirname $opt_war_dir`
17:             ansible test -m file -a "path=$opt_local_repo/$dir_path state=directory"
18:         ansible test -m copy -a "src=$opt_local_repo/$opt_war_dir dest=$opt_local_repo/$opt_war_dir"
19:             if [ "$opt_version"x != "break"x ];then
20:               #将备份文件记录发送到目标目录
21:                 ansible test -m copy -a "src=$opt_info_dir/ dest=$opt_info_dir"
22:                 #将备份文件发送到目标目录
23:                 ansible test -m copy -a "src=$opt_bak_dir/$opt_version/ dest=$opt_bak_dir/$opt_version"
24:             fi
25:             #目标开始进行环境部署
26:           ansible test -m script -a "/data/project/test/script/sw_build.sh --local-repository=$opt_local_repo --webdir=$opt_web_dir --package=$opt_war_dir"
27:         else
28:           echo "project exit"
29:         fi
30:         ;;
31:     rollback)
32:         echo "rollback: $deploy_envirenment"
33:         ansible test -m script -a "/data/project/test/script/sw_rollback.sh --backup-dir=$opt_bak_dir --webdir=$opt_web_dir --infodir=$opt_info_dir --version=$opt_version --package=$opt_war_dir"
34:         ;;
35:     *)
36:     exit
37:         ;;
38: esac

pipeline

  • 这种方式是jenkins 推荐的方式,对其支持的也比较好,在终端页面,能看到其执行的状态,并可以查看日志. 其配置语言类似groovy,官网有很多介绍,java build demo 这个是一个Java编译demo,可以参考下.
  • mvn 打包,总是从中央仓库下载jar
  • Jenkinsfile 调用shell如果权限被拒的话,将本地脚本添加执行权限eg: chmod +x ./shell.sh
