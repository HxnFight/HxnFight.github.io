elements

thread group

  • 此选项为必须选项,用来模拟用户发送的请求,controll,sampler 必须放置于其内
  • 配置并发有两种模式
    • 线程数量,上升时间,执行次数
    • 定时器

controller

  • 向服务器发送请求,是测试逻辑的主要部分
  • sampler
    • 简单的发送请求,可以组合多个
    • 用assertion 来过滤结果
    • 用listener来查看结果
    • 此controll是顺序执行
  • logic controller
    • 根据一定的条件来执行,所以顺序可能不一致
    • 可以测试web登录

timers

  • 在测试组之间产生延迟,以免压垮程序

assertions

  • 可以增加断言,来判断数据的正确性

configuration

  • 添加配置的入口,这里注意其作用范围

listeners

  • 监听输出结果,用于产生测试报告

pre-processor elements

  • 在请求之前做的额外处理
  • 加密接口的处理方式
    • beanshell 来解决加密访问
    • groovy 方式
    • 这两种方式均可,只看对那种语言熟悉
    • 此外还介绍了其他的脚本语言,可以参考官方文档

post-processor elements

  • 在请求之后做的额外处理

execution order

  • Configuration elements
  • Pre-Processors
  • Timers
  • Sampler
  • Post-Processors (unless SampleResult is null)
  • Assertions (unless SampleResult is null)
  • Listeners (unless SampleResult is null)

scoping rules

  • 各个组件的有效范围
